Guía de Creación de Claves

Simétricas (AES-256-GCM):

  • Más rápidas y eficientes
  • Para encriptar datos hasta 4KB directamente
  • Para datos más grandes, usar "claves de datos"
  • Soportan rotación automática

Asimétricas (RSA, ECC):

  • Para firmas digitales y verificación
  • Para criptografía de clave pública
  • No soportan rotación automática
  • Más lentas pero más seguras para ciertos casos

Los aliases son nombres amigables para tus claves:

  • Deben comenzar con alias/
  • Solo letras, números, guiones, barras y guiones bajos
  • Pueden cambiarse sin afectar la funcionalidad
  • Útiles para identificar claves en aplicaciones

La rotación mantiene la seguridad de tus datos:

  • Automática: AWS rota la clave cada año
  • Manual: Puedes rotar cuando necesites
  • Los datos encriptados con la clave anterior siguen siendo accesibles
  • Recomendado para claves de encriptación activa
